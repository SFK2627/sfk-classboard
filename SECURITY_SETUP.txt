SFK CLASSBOARD SECURITY SETUP
================================

Do these steps before publishing the new Firestore rules.

1. ENABLE FIREBASE EMAIL/PASSWORD LOGIN

Firebase Console
> Authentication
> Sign-in method
> Email/Password
> Enable
> Save


2. CREATE THE TWO LOGIN ACCOUNTS

Firebase Console
> Authentication
> Users
> Add user

Create:

Admin account
Email: admin@sfk-classboard.app
Password: your NEW Admin PIN/password (minimum 6 characters)

Shared Officer account
Email: officers@sfk-classboard.app
Password: your NEW shared Officer PIN/password (minimum 6 characters)

All officers will enter the same Officer PIN. The email is handled silently by
the ClassBoard and does not appear on the login form.

Do not reuse the old exposed PINs.


3. PUBLISH THE FIRESTORE RULES

Firebase Console
> Firestore Database
> Rules

Replace the rules with FIREBASE_RULES.txt, then click Publish.


4. UPDATE AND REDEPLOY APPS SCRIPT

Replace the Apps Script Code.gs with the included Code.gs.

In Apps Script:
> Project Settings
> Script Properties
> Add script property

Property: FIREBASE_WEB_API_KEY
Value: the apiKey value from firebase-config.js

Then:
> Deploy
> Manage deployments
> Edit
> Version: New version
> Deploy

The existing web app URL can stay the same.


5. UPLOAD THE WEBSITE FILES

Upload/replace every file included in the security ZIP.


FIREBASE STORAGE IS NOT REQUIRED

Do not enable Firebase Storage and do not add a payment method for this update.
The current ClassBoard sends Memory and announcement files to Google Drive
through Apps Script. Only Firebase Authentication, Firestore, and Apps Script
are required.


WHAT CHANGED

- No Admin or Officer PIN is stored in public JavaScript.
- No PIN is saved in localStorage or sessionStorage.
- Firebase Authentication validates the login.
- Parents and students retain public read-only access.
- Only authenticated Admin/Officer accounts can write to Firestore.
- Only Admin can permanently delete records.
- Apps Script uploads validate the Firebase login token.
- The shared Officer PIN remains supported through one shared Officer account.
